Featured Project

PM360 Project
Management Tool

A comprehensive project tracking solution for 45+ active infrastructure projects,
reducing project delays by 30% and providing real-time visibility for leadership.

The Problem

The organisation was managing 45+ active infrastructure projects simultaneously, but there was no centralised system to track progress, milestones, or bottlenecks. Key challenges included:

  • Project data scattered across spreadsheets, emails, and multiple systems
  • No visibility into project status for leadership
  • Missed milestones and delayed go-live dates
  • Inefficient handoffs between requirements, BPR, UAT, and training phases
  • Difficulty identifying which projects were at risk
  • No standardised project management methodology

My Role

I led the design, development, and implementation of the PM360 tool. My responsibilities included:

  • Requirements Gathering: Interviewed project managers, technical leads, and leadership to define requirements
  • Tool Design: Designed the project tracking framework and user interface
  • Development: Built the tool using ServiceNow and Power BI
  • Milestone Definition: Defined standard project phases and milestone templates
  • Dashboard Creation: Developed real-time dashboards for project visibility
  • User Training: Trained project managers on using the tool

Approach & Methodology

Step 1: Requirements Gathering

Conducted workshops with key stakeholders to understand their needs:

  • Project Managers: Needed milestone tracking, task assignment, and risk logging
  • Technical Leads: Required visibility into technical scope and UAT status
  • Leadership: Wanted high-level dashboards showing project health and at-risk projects

Step 2: Project Lifecycle Definition

Standardised the project lifecycle into 5 key phases:

Phase 1: Requirements Gathering
Business requirements, functional specifications, stakeholder sign-off
Phase 2: Business Process Reengineering (BPR)
Process mapping, gap analysis, to-be process design
Phase 3: Technical Scope & Development
Technical design, development, system configuration
Phase 4: User Acceptance Testing (UAT)
Test planning, execution, defect tracking, user sign-off
Phase 5: Training & Go-Live
User training, documentation, deployment, hypercare

Step 3: Tool Development

Built the PM360 tool with the following features:

  • Project Dashboard: Centralised view of all 45+ projects with status indicators
  • Milestone Tracker: Real-time tracking of milestone completion across phases
  • Risk Register: Centralised logging and tracking of project risks
  • Document Repository: Centralised storage for all project documents
  • Automated Notifications: Alerts for upcoming milestones and overdue tasks
  • Resource Management: Tracking of resource allocation across projects

Step 4: Dashboard Development

Created Power BI dashboards for different stakeholder groups:

  • Leadership Dashboard: Project health overview, at-risk projects, milestone completion trends
  • Project Manager Dashboard: Individual project status, task lists, upcoming milestones
  • Resource Dashboard: Resource allocation, utilisation, availability

Step 5: Implementation & Training

Rolled out the tool in phases:

  • Pilot with 5 projects to test and refine
  • Training sessions for all project managers and key users
  • Full rollout to all 45+ projects
  • Ongoing support and continuous improvement
Technologies Used:
ServiceNow Power BI SQL Agile Methodology Jira Flow Designer

Challenges Overcome

  • Standardisation: Different project managers had different ways of tracking projects. Created standardised templates while allowing flexibility.
  • Data Migration: Migrated project data from multiple spreadsheets and systems into the new tool.
  • User Adoption: Some project managers were resistant to change. Addressed through training, support, and demonstrating clear benefits.
  • Real-time Updates: Ensured dashboards reflected real-time data without performance issues across 45+ projects.

Outcome & Impact

  • Project delays reduced by 30% through better milestone tracking and early risk identification
  • 100% visibility into project status for leadership
  • Standardised project management methodology across all 45+ projects
  • Improved resource allocation through centralised resource tracking
  • Reduced administrative overhead by eliminating manual status reporting
  • Faster issue resolution through centralised risk and issue tracking

Key Learnings

This project taught me that standardisation doesn't mean rigidity. While having a standard project lifecycle is valuable, different projects have different needs. Building flexibility into the tool while maintaining standard reporting was critical to adoption.

I also learned that visibility alone is powerful. Simply giving leadership real-time visibility into project status helped them identify and address bottlenecks earlier, reducing delays without any other changes.
