Featured Project

SAP-ServiceNow Integration
& Data Migration

Led data migration for SAP-ServiceNow integration across the gas business,
transferring over 500,000 assets and 1.2 million transaction records with 90% data accuracy.

The Problem

The organisation needed to integrate SAP and ServiceNow to create a unified IT service management and enterprise resource planning ecosystem. Key challenges included:

  • Siloed systems with no data synchronisation between SAP and ServiceNow
  • Manual data entry across both systems causing inconsistencies and errors
  • Over 500,000 assets tracked in SAP needed to be accessible in ServiceNow CMDB
  • 1.2 million transaction records requiring migration with complete integrity
  • No automated process for asset updates between systems
  • Risk of business disruption during migration
  • Need for real-time synchronisation post-integration

My Role

I led the end-to-end data migration and integration project. My responsibilities included:

  • Migration Planning: Developed data migration strategy and timeline
  • Data Mapping: Mapped data fields between SAP and ServiceNow
  • Data Cleansing: Led data quality assessment and cleansing efforts
  • Integration Design: Designed real-time synchronisation architecture
  • Migration Execution: Managed phased migration of 500,000+ assets
  • Validation: Implemented data validation and accuracy checks
  • Post-Migration Support: Ensured business continuity after migration

Data Types Migrated

Asset Master Data (500,000+ records) Transaction Records (1.2M+ records) Vendor Information Material Codes GL Accounts Cost Centres Purchase Orders Invoices Maintenance History Contract Data

Approach & Methodology

Step 1: Discovery & Assessment

Conducted comprehensive assessment of both systems:

  • Analysed SAP data structures, tables, and relationships
  • Reviewed ServiceNow CMDB schema and requirements
  • Identified data quality issues in source system
  • Documented integration requirements and success criteria
  • Assessed risk factors and mitigation strategies

Step 2: Data Mapping & Transformation

Mapped data fields between SAP and ServiceNow:

  • Created detailed field mapping document for all data types
  • Defined transformation rules for data format conversion
  • Handled data type mismatches between systems
  • Established reference data mapping for codes and categories
  • Documented business rules for data synchronisation

Step 3: Data Cleansing & Preparation

Led data quality improvement initiatives:

  • Identified and removed duplicate records
  • Standardised data formats across all records
  • Validated mandatory fields for completeness
  • Corrected inconsistent reference data
  • Archived obsolete and inactive records

Step 4: Integration Architecture Design

Designed real-time integration architecture:

REST API Integration: ServiceNow REST APIs for real-time data exchange
Mid Server Configuration: ServiceNow Mid Server for secure on-premise SAP connectivity
Web Services: SOAP web services for batch data synchronisation
Error Handling: Robust error logging and retry mechanisms
Data Validation: Pre- and post-sync validation checks

Step 5: Phased Migration Execution

Executed migration in phases to minimise risk:

  • Phase 1: Pilot migration with 1,000 assets to validate process
  • Phase 2: Migration of non-critical asset data
  • Phase 3: Migration of transaction records
  • Phase 4: Migration of critical asset data
  • Phase 5: Final validation and reconciliation

Step 6: Validation & Reconciliation

Implemented comprehensive validation:

  • Record count reconciliation between source and target
  • Sample-based data accuracy verification
  • Critical field validation for all records
  • Business user acceptance testing
  • Post-migration data quality monitoring
Technologies Used:
SAP ServiceNow REST APIs SOAP Web Services Data Governance Mid Server ETL Tools

Challenges Overcome

  • Data Volume: Migrating 500,000+ assets and 1.2M transactions required careful planning and batch processing to avoid performance issues.
  • Data Quality: Source data had significant quality issues including duplicates, missing values, and inconsistent formats. Addressed through systematic cleansing.
  • System Availability: Migrations needed to complete during maintenance windows. Developed efficient processes to work within time constraints.
  • Field Mapping Complexity: SAP and ServiceNow have different data models. Extensive mapping and transformation logic required.
  • Real-time Sync: Ensuring real-time synchronisation without performance degradation required careful API design and optimisation.
  • Business Continuity: Ensured no disruption to business operations during migration through phased approach and fallback planning.

Outcome & Impact

  • 90% data accuracy achieved post-migration
  • Seamless integration preventing business disruption
  • 500,000+ assets successfully migrated to ServiceNow CMDB
  • 1.2M transaction records migrated with complete integrity
  • Real-time synchronisation established between SAP and ServiceNow
  • Eliminated manual data entry between systems
  • Improved data governance with single source of truth
  • Enhanced reporting accuracy across both systems

Key Learnings

This project taught me that data migration success depends more on preparation than execution. Investing time in data cleansing, validation, and pilot testing prevented major issues during full migration.

I also learned that stakeholder communication is critical. Regular updates on migration progress, risks, and milestones kept everyone aligned and reduced anxiety about potential disruptions.

Finally, having a rollback plan gave confidence to proceed. Knowing we could revert if something went wrong allowed us to move forward decisively.
