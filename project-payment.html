Featured Project

Payment Tracking &
Cash Flow Optimization System

An automated payment tracking application monitoring £8.5M in outstanding receivables,
reducing collection time from 95 to 52 days and improving cash flow by £1.2M.

The Problem

The organisation had £8.5M in outstanding receivables from government authorities for project collaterals and regulatory deposits. The manual follow-up process was causing significant cash flow delays and administrative burden.

Key challenges included:

  • Manual tracking using spreadsheets across multiple departments
  • No visibility into payment status or collection progress
  • Missed follow-ups leading to delayed payments
  • Average collection time of 95 days
  • Late payment penalties accumulating over time
  • Finance leadership lacked real-time visibility into receivables

My Role

I led the end-to-end development of the payment tracking system. My responsibilities included:

  • Process Analysis: Mapped the existing manual collection process and identified bottlenecks
  • Solution Design: Designed automated workflow architecture using ServiceNow and Power Automate
  • Dashboard Development: Built real-time Power BI dashboard for finance leadership
  • SAP Integration: Integrated with SAP FICO for accurate reconciliation
  • User Training: Trained finance teams on the new system

Approach & Methodology

Step 1: Process Mapping

Documented the entire payment collection process, from invoice generation to final settlement. Identified key bottlenecks: manual follow-up scheduling, lack of automated reminders, no escalation workflow, and fragmented data across spreadsheets.

Step 2: Solution Design

Designed an automated payment tracking system with:

  • Centralised database for all receivable records
  • Automated follow-up workflows based on payment due dates
  • Escalation rules for overdue payments
  • Real-time dashboard for visibility
  • Integration with SAP FICO for financial reconciliation

Step 3: Workflow Automation

Using ServiceNow Flow Designer and Power Automate, I built:

  • Automated Reminders: Email notifications at 30, 15, 7, and 1 day before due date
  • Escalation Workflow: Automatic escalation to senior management for payments overdue by 30+ days
  • Status Tracking: Real-time updates on payment status for each receivable
  • Document Management: Centralised storage for all supporting documents

Step 4: Dashboard Development

Created a real-time Power BI dashboard showing:

  • Total outstanding receivables by authority
  • Aging analysis (0-30 days, 31-60 days, 61-90 days, 90+ days)
  • Collection trends over time
  • Expected cash inflow forecast
  • Department-wise collection performance

Step 5: SAP Integration

Integrated the tracking system with SAP FICO to ensure:

  • Automatic reconciliation of payments received
  • Real-time update of receivable status
  • Accurate financial reporting
  • Audit trail for all transactions
Technologies Used:
SAP FICO Power BI Power Automate ServiceNow Flow Designer SQL Excel

Challenges Overcome

  • Data Fragmentation: Payment data was spread across multiple spreadsheets and systems. Consolidated all data into a single source of truth.
  • System Integration: Successfully integrated ServiceNow with SAP FICO despite complex authentication and data mapping requirements.
  • User Adoption: Finance teams were accustomed to manual processes. Conducted training sessions and created user guides to ensure smooth transition.
  • Data Accuracy: Implemented validation rules to ensure data quality before processing automated workflows.
  • Regulatory Compliance: Ensured all automated processes complied with financial regulations and audit requirements.

Outcome & Impact

  • Collection time reduced from 95 to 52 days — a 45% improvement
  • £1.2M cash flow improvement through faster collections
  • Late payment penalties eliminated through automated follow-ups
  • 100% visibility into receivables status for finance leadership
  • Reduced manual effort by 80% for the finance team
  • Improved audit compliance with complete transaction history

Key Learnings

This project reinforced the importance of end-to-end process automation. Manual processes not only consume time but also introduce errors and delays. By automating follow-ups and providing real-time visibility, we transformed a reactive, stressful process into a proactive, manageable one.

I also learned that system integration is often the biggest challenge. Investing time upfront to understand data structures and authentication requirements paid off significantly during implementation.
